# গ্রিন হেভেন রিভিউ সিস্টেম — সেটআপ গাইড (বাংলা)

এই গাইডটি অনুসরণ করলে আপনার গ্রাহক (অভিভাবক ও ছাত্রছাত্রীরা) **Google / Facebook**
দিয়ে লগইন করে আপনার ওয়েবসাইটে রিভিউ দিতে পারবে, এবং আপনি **অনুমোদন (approve)**
দেওয়ার পর সেই রিভিউ [reviews.html](reviews.html) পেজে দেখাবে।

> 🕒 সময় লাগবে আনুমানিক ২০–৩০ মিনিট। শুধু একবারই করতে হবে।
> খরচ: সম্পূর্ণ **ফ্রি** (Firebase Spark প্ল্যান যথেষ্ট)।

---

## যা যা তৈরি হয়েছে
| ফাইল | কাজ |
|------|------|
| `reviews.html` | রিভিউ পেজ + "আপনার রিভিউ দিন" ফর্ম (পরিবর্তন করা হয়েছে) |
| `admin.html` | আপনার জন্য মডারেশন প্যানেল (রিভিউ অনুমোদন/বাতিল) |
| `firebase-config.js` | ⚠️ এখানে আপনার Firebase কী বসাতে হবে |
| `firestore.rules` | নিরাপত্তা নিয়ম (Firebase Console-এ পেস্ট করতে হবে) |

---

## ধাপ ১ — Firebase প্রজেক্ট তৈরি করুন
1. ব্রাউজারে যান 👉 https://console.firebase.google.com
2. আপনার Google অ্যাকাউন্ট দিয়ে লগইন করুন।
3. **"Add project" / "প্রজেক্ট যোগ করুন"** ক্লিক করুন।
4. নাম দিন, যেমন: `green-heaven-reviews` → **Continue**।
5. Google Analytics দরকার নেই — বন্ধ (disable) করে **Create project** চাপুন।

---

## ধাপ ২ — ওয়েব অ্যাপ যোগ করে কী (key) সংগ্রহ করুন
1. প্রজেক্ট খোলার পর হোম স্ক্রিনে **`</>` (Web)** আইকনে ক্লিক করুন।
2. App nickname দিন (যেমন `green-heaven-web`) → **Register app**।
3. একটি কোড ব্লক দেখাবে `const firebaseConfig = { ... }`। এর ভেতরের মানগুলো লাগবে।
4. **`firebase-config.js`** ফাইল খুলে নিচের অংশটি আপনার মান দিয়ে রিপ্লেস করুন:

```js
const firebaseConfig = {
  apiKey: "এখানে-আপনার-apiKey",
  authDomain: "your-project-id.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project-id.appspot.com",
  messagingSenderId: "এখানে-আপনার-senderId",
  appId: "এখানে-আপনার-appId"
};
```

> ℹ️ এই কী-গুলো গোপন নয় — ওয়েবে এগুলো পাবলিক থাকাই স্বাভাবিক।
> আসল নিরাপত্তা আসে ধাপ ৬-এর সিকিউরিটি রুল থেকে।

---

## ধাপ ৩ — Google লগইন চালু করুন
1. বাঁ পাশের মেনু থেকে **Build → Authentication** → **Get started**।
2. **Sign-in method** ট্যাব → **Google** নির্বাচন করুন।
3. **Enable** টগল অন করুন → support email বাছাই করুন → **Save**।

✅ Google লগইন প্রস্তুত।

---

## ধাপ ৪ — Facebook লগইন চালু করুন
> একটু সময়সাপেক্ষ, কিন্তু একবারই করতে হবে।

**৪ক) Facebook অ্যাপ তৈরি:**
1. যান 👉 https://developers.facebook.com/apps → **Create App**।
2. Use case: **Authenticate and request data from users with Facebook Login** → Next।
3. App type: **Consumer / Other** → অ্যাপের নাম দিন → তৈরি করুন।
4. ড্যাশবোর্ডে বাঁ মেনুতে **Facebook Login → Settings** খুলুন।

**৪খ) Firebase ও Facebook সংযোগ:**
1. Firebase Console → **Authentication → Sign-in method → Facebook** → Enable।
2. এখানে **App ID** ও **App Secret** চাইবে — এ দুটি Facebook ড্যাশবোর্ডের
   **Settings → Basic** থেকে কপি করে বসান।
3. Firebase-এ একটি **OAuth redirect URI** দেখাবে (যেমন
   `https://your-project-id.firebaseapp.com/__/auth/handler`) — সেটি **কপি** করুন।
4. Facebook ড্যাশবোর্ড → **Facebook Login → Settings** → **Valid OAuth Redirect URIs**
   বক্সে ওই লিংকটি পেস্ট করে **Save changes**।
5. Firebase-এ ফিরে **Save**।
6. Facebook অ্যাপটি **Live/Public** মোডে দিন (ড্যাশবোর্ডের উপরে টগল), নাহলে শুধু
   আপনি নিজে লগইন করতে পারবেন।

✅ Facebook লগইন প্রস্তুত।

---

## ধাপ ৫ — Firestore ডেটাবেস তৈরি করুন
1. বাঁ মেনু → **Build → Firestore Database** → **Create database**।
2. **Start in production mode** বাছুন → Next।
3. লোকেশন বাছুন (যেমন `asia-south1` — কাছাকাছি ও ভালো গতি) → **Enable**।

---

## ধাপ ৬ — নিরাপত্তা নিয়ম (Rules) বসান
1. Firestore Database → উপরের **Rules** ট্যাবে যান।
2. সেখানে যা লেখা আছে সব মুছে, এই প্রজেক্টের **`firestore.rules`** ফাইলের
   পুরো লেখাটি কপি করে পেস্ট করুন।
3. **Publish** ক্লিক করুন।

এই নিয়ম নিশ্চিত করে:
- যে কেউ শুধু **অনুমোদিত** রিভিউ দেখতে পারবে।
- লগইন করা ব্যবহারকারী শুধু নিজের নামে রিভিউ জমা দিতে পারবে।
- শুধু **আপনি (অ্যাডমিন)** রিভিউ অনুমোদন/বাতিল করতে পারবেন।

---

## ধাপ ৭ — নিজেকে অ্যাডমিন বানান (খুব জরুরি)
আপনি যেন রিভিউ অনুমোদন করতে পারেন, তার জন্য আপনার অ্যাকাউন্টকে অ্যাডমিন বানাতে হবে।

1. ওয়েবসাইটের **`admin.html`** পেজটি খুলুন (যে অ্যাকাউন্ট দিয়ে অনুমোদন করতে চান
   সেটি দিয়ে **Google/Facebook লগইন** করুন)।
2. পেজে দেখাবে **"আপনি এখনো অ্যাডমিন নন"** এবং আপনার একটি **UID** দেখাবে — **কপি** করুন।
3. Firebase Console → **Firestore Database → Data** ট্যাব।
4. **Start collection** → Collection ID লিখুন হুবহু: `admins` → Next।
5. **Document ID** ঘরে আপনার কপি করা **UID** পেস্ট করুন।
6. একটি ফিল্ড যোগ করুন (যেকোনো নাম, যেমন Field `role` = `admin`) → **Save**।
7. **`admin.html`** পেজটি রিফ্রেশ করুন — এখন মডারেশন প্যানেল দেখাবে! ✅

> একাধিক অ্যাডমিন চাইলে প্রত্যেকের UID দিয়ে `admins` কালেকশনে আলাদা ডকুমেন্ট যোগ করুন।

---

## ধাপ ৮ — ডোমেইন অনুমোদন করুন (হোস্ট করার সময়)
আসল ডোমেইনে (যেমন `greenheavensuperhostel.com`) লগইন কাজ করতে হলে:
1. Firebase Console → **Authentication → Settings → Authorized domains**।
2. **Add domain** → আপনার ডোমেইন লিখুন (যেমন `greenheavensuperhostel.com`)।

> `localhost` আগে থেকেই অনুমোদিত, তাই কম্পিউটারে টেস্ট করতে কিছু লাগবে না।

---

## ধাপ ৯ — রিভিউ দেখানোর জন্য Index তৈরি (একবার)
প্রথমবার রিভিউ অনুমোদন করার পর reviews পেজে দেখাতে একটি ছোট index লাগে।
- `admin.html`-এ একটি রিভিউ **অনুমোদন** করুন।
- এরপর ব্রাউজারে **F12 → Console** খুললে একটি লিংকসহ মেসেজ আসতে পারে
  ("create index ...")। সেই **লিংকে ক্লিক করে → Create Index** চাপুন (১–২ মিনিট লাগে)।
- এরপর থেকে অনুমোদিত রিভিউ স্বয়ংক্রিয়ভাবে reviews পেজে দেখাবে।

> Index তৈরি না হওয়া পর্যন্ত পেজের পুরোনো (ডেমো) রিভিউগুলো ঠিকঠাক দেখাবে — কিছু ভাঙবে না।

---

## ✅ কীভাবে চলবে (সারসংক্ষেপ)
1. গ্রাহক [reviews.html](reviews.html)-এ **"আপনার রিভিউ দিন"** চাপে।
2. Google/Facebook দিয়ে লগইন করে স্টার + মতামত দিয়ে জমা দেয়।
3. রিভিউ **pending** অবস্থায় জমা হয় (এখনো কেউ দেখে না)।
4. আপনি **`admin.html`**-এ গিয়ে দেখে **অনুমোদন** দেন।
5. অনুমোদিত রিভিউ সাথে সাথে রিভিউ পেজে সুন্দর কার্ড আকারে দেখায়। 🎉

## ❓ সমস্যা হলে
- **লগইন পপআপ আসে না / বন্ধ হয়ে যায়** → ব্রাউজারে পপআপ ব্লক করা আছে কিনা দেখুন;
  হোস্ট করা ডোমেইন **Authorized domains**-এ যোগ করা আছে কিনা দেখুন (ধাপ ৮)।
- **রিভিউ জমা হয় না** → `firebase-config.js`-এ কী ঠিক বসেছে কিনা এবং নিয়ম (ধাপ ৬)
  Publish হয়েছে কিনা দেখুন।
- **অ্যাডমিন প্যানেল খোলে না** → ধাপ ৭ আবার দেখুন; UID হুবহু ঠিক আছে কিনা নিশ্চিত করুন।
- **অনুমোদিত রিভিউ পেজে আসে না** → ধাপ ৯ (Index) সম্পন্ন করুন।

প্রয়োজনে আপনার ডেভেলপারকে এই গাইডটি দেখান — তিনি সহজেই সম্পন্ন করতে পারবেন।
